# API Contracts

Base URL prefix: `/api/v1` (`src/main.ts:9`). Swagger UI is mounted at `/api/v1/docs` (`src/config/swagger.config.ts:14-18`).

Global request behavior:

- JSON body DTO validation is strict: unknown fields are forbidden and stripped by `ValidationPipe` (`src/main.ts:11-17`).
- Authenticated routes use `Authorization: Bearer <accessToken>` extracted exactly as `Bearer` (`src/common/guards/jwt-auth.guard.ts:46-49`).
- `IdParamDto.id` path params must be UUIDs when controllers use `@Param() params: IdParamDto` (`src/common/dto.ts:24-27`).

Common error responses from code:

- `401 Missing bearer token` when bearer token is absent (`src/common/guards/jwt-auth.guard.ts:25-27`).
- `401 Invalid or expired token` when JWT verification fails (`src/common/guards/jwt-auth.guard.ts:29-43`).
- Role guard returns false for missing roles, which Nest treats as forbidden (`src/common/guards/roles.guard.ts:11-24`).
- Validation errors are generated by Nest validation pipe for DTO violations (`src/main.ts:11-17`).

## Auth

### POST `/api/v1/auth/register`

Authentication Required: No.

Purpose: Create a public student account and return a JWT access token.

Request Body:

```json
{
  "fullName": "Adamu Muhammad",
  "email": "adamu@example.com",
  "password": "password123",
  "role": "STUDENT"
}
```

Validation:

- `fullName`: string, not empty (`src/common/dto.ts:30-34`).
- `email`: email (`src/common/dto.ts:36-38`).
- `password`: string, not empty (`src/common/dto.ts:40-43`).
- `role`: optional `UserRole` enum (`src/common/dto.ts:45-48`).

Business Rules:

- Public registration supports only `STUDENT`; `MENTOR` or `ADMIN` returns bad request (`src/auth/auth.service.ts:24-29`).
- Email is checked before create; duplicate returns conflict (`src/auth/auth.service.ts:31-34`).
- Email is stored lowercase (`src/auth/auth.service.ts:37-41`).
- Password is bcrypt-hashed with cost 12 (`src/auth/auth.service.ts:36`).

Success Response:

```ts
{
  user: SafeUser;
  accessToken: string;
}
```

`passwordHash` is removed by `toSafeUser()` (`src/users/user.presenter.ts:3-8`).

Error Responses:

- `400 Public registration only supports students`.
- `409 Email is already registered`.
- Validation errors.

Frontend Notes:

- Do not expose role selection for public registration unless only `STUDENT` can be selected.
- Store `accessToken`; there is no refresh token or cookie session.

